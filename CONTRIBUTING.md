# Contributing to KoalaData

Thank you for helping improve privacy-respecting extension analytics.

## Before opening an issue

- Search existing issues and the changelog.
- Use the appropriate issue template.
- Describe expected and actual behavior with reproducible steps.
- Remove usernames, session cookies, raw CSV contents, access logs, and other personal or secret data.
- Report vulnerabilities privately according to [SECURITY.md](SECURITY.md).

## Development setup

Requirements: Node.js 22, npm, and optionally Docker.

```bash
npm ci
npx drizzle-kit push
npm run dev
```

Use synthetic CSV samples in tests. Never commit real Chrome Web Store exports.

## Pull requests

1. Keep each pull request focused on one coherent change.
2. Explain the user-facing behavior and privacy impact.
3. Add or update tests for changed behavior.
4. Update documentation and `docs/CHANGELOG.md` when appropriate.
5. Run the verification commands below.

```bash
npm run check
npm run test:unit -- --run
npm run test:e2e
npm run build
git diff --check
```

For dependency or deployment changes, also run:

```bash
npm audit --audit-level=low
docker build --tag koaladata:local .
```

## Code expectations

- Preserve server-side authorization checks; hiding a UI control is not authorization.
- Treat uploaded files, URLs, filenames, and CSV cells as untrusted input.
- Keep passwords, tokens, cookies, private exports, and precise network identifiers out of logs.
- Maintain keyboard access, semantic landmarks, color contrast, and responsive behavior.
- Keep user-facing metric names truthful to their source and aggregation semantics.
- Include every supported locale when changing shared localized behavior.

## Database changes

- Add a versioned migration under `migrations/`.
- Keep migrations forward-only and safe for existing installations.
- Update schema snapshots when generated by Drizzle.
- Test both a new database and an upgraded database.
